# Deploying to Render

## Prerequisites
- GitHub repository with your code pushed
- Render account (free at https://render.com)

## Steps to Deploy

### 1. Connect GitHub
1. Go to https://render.com/dashboard
2. Click "New +" → "Web Service"
3. Connect your GitHub account and select this repository
4. Choose branch: `main`

### 2. Configure
- **Name**: `maybankpdf2json-web` (or your preferred name)
- **Region**: Choose the closest to your users
- **Runtime**: Python
- **Build Command**: Already defined in render.yaml
- **Start Command**: Already defined in render.yaml

### 3. Environment Variables (Set in Render dashboard)
Render will auto-generate these from render.yaml:
- `DEBUG`: `False` (production mode)
- `SECRET_KEY`: Auto-generated by Render
- `ALLOWED_HOSTS`: Auto-set to your Render domain
- `CSRF_TRUSTED_ORIGINS`: Auto-set to https://your-app.onrender.com

### 4. Database
- Render will create a free PostgreSQL database (specified in render.yaml)
- Connection string auto-injected as `DATABASE_URL` environment variable

### 5. Deploy
1. Click "Create Web Service"
2. Render automatically builds and deploys
3. Wait for "Your service is live" message
4. Click the URL to test

## Post-Deployment

### First Time
- Database migrations run automatically (build.sh)
- Static files collected automatically

### Updates
Push to GitHub → Render auto-redeploys

### Logs
View real-time logs in Render dashboard

## Troubleshooting

**Deployment failed?**
- Check build logs in Render dashboard
- Ensure `requirements.txt` has all dependencies
- Verify `build.sh` is executable

**Static files not loading?**
- They're served by WhiteNoise (configured in settings.py)
- No separate static file server needed

**Database issues?**
- Check DATABASE_URL environment variable is set
- Run migrations manually if needed from Render shell

## Costs
- **Free tier**: 0.5 GB RAM, 1 GB storage, free PostgreSQL database
- After 15 minutes of inactivity, web service goes to sleep (wakes on request)
- Perfect for development/low-traffic projects

## Switching to PostgreSQL (optional locally)

Install PostgreSQL client:
```bash
pip install psycopg2-binary
```

Create `.env` with:
```
DATABASE_URL=postgresql://user:password@localhost/maybankpdf2json
```

Then:
```bash
python manage.py migrate
```
